# 抠图功能实现计划

## 需求概述

实现用户可控的抠图功能，支持自定义抠图区域，提供预览和多种导出选项。

## 涉及修改的文件

1. `emo-j/src/utils/imageProcessor.ts` - 添加新的分割和抠图函数
2. `emo-j/src/components/ExportPanel/ExportPanel.tsx` - 主要UI和逻辑修改
3. `emo-j/src/components/ExportPanel/ExportPanel.css` - 添加新的样式
4. `CLAUDE.md` - 更新项目文档

## 实现步骤

### 第一步：修改 imageProcessor.ts

**新增功能**：
1. 添加 `sliceImageOnly()` 函数 - 仅分割图片不进行抠图处理
2. 添加 `cropImageWithBounds()` 函数 - 支持自定义区域的抠图
3. 导出的类型需要扩展以支持区分"原始分割"和"抠图后"

**新增类型**：
```typescript
export interface SliceResult {
  blob: Blob;
  index: number;
  isProcessed: boolean; // 标记是否经过抠图处理
}
```

### 第二步：修改 ExportPanel.tsx

**新增状态**：
1. `mattingEnabled` - 抠图功能开关状态
2. `mattingSettings` - 记录每个子图的抠图设置 (Map<index, bounds | null>)
3. `originalSlices` - 原始分割结果（未抠图）
4. `isBatchMode` - 当前是否为批量设置模式
5. `activeSliceIndex` - 当前正在设置抠图区域的子图索引

**新增函数**：
1. `handleToggleMatting()` - 切换抠图功能开关
2. `handleSliceBoundsChange(index, bounds)` - 更新单个子图的抠图区域
3. `handleBatchSetBounds()` - 批量为所有子图设置抠图区域
4. `handleClearBounds(index)` - 清除单个子图的抠图区域设置
5. `handleClearAllBounds()` - 清除所有子图的抠图区域设置
6. `generateOriginalPreviews()` - 生成原始分割预览（不抠图）
7. `handleExportAllMatting()` - 一键导出所有抠图图像
8. `handleExportAllOriginal()` - 一键导出所有原始分割图像
9. `getExportResult(index)` - 根据设置决定导出哪张图片

### 第三步：修改 ExportPanel.css

**新增样式**：
1. `.matting-toggle` - 抠图开关样式
2. `.matting-settings-panel` - 抠图设置面板
3. `.batch-actions` - 批量操作按钮组
4. `.slice-item` - 子图项（包含原始/抠图状态标记）
5. `.has-matting-indicator` - 标记是否已设置抠图
6. `.original-preview` / `.matting-preview` - 预览状态标记

### 第四步：更新 CLAUDE.md

在"项目进度"和"更新日志"部分记录此功能。

## UI 交互流程

### 1. 功能开关
- ExportPanel 顶部添加"启用抠图"开关
- 关闭时：只显示原始分割预览，一键导出为原始图像
- 开启时：显示抠图相关操作界面

### 2. 区域选择模式
- 点击子图进入单独设置模式
- 提供"批量设置所有"按钮
- 每个子图显示是否已设置抠图区域的视觉标记

### 3. 预览功能
- 原始分割：显示灰色占位背景的分割图
- 抠图预览：显示透明背景的抠图效果
- 两种预览可切换查看

### 4. 导出规则
- 已设置抠图区域的子图：点击下载 → 导出抠图后图像
- 未设置抠图的子图：点击下载 → 导出原始分割图像
- 一键导出时自动按上述规则处理

### 5. 快捷导出选项
- "一键导出抠图图像" - 仅导出已设置抠图的图像
- "一键导出原始图像" - 导出所有原始分割图像
