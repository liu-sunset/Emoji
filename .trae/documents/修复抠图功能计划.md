# 修复抠图功能计划

## 任务概述

1. 删除单个图片的抠图功能，只保留批量抠图功能
2. 修复抠图后预览无法正常显示的问题

## 问题分析

### 问题1：单个抠图功能需要删除
当前代码支持两种抠图方式：
- 批量自动抠图（保留）
- 单个图片点击选择区域抠图（删除）

### 问题2：预览显示问题
从截图可以看到，表情1和表情2的位置显示为黑色，没有正确显示图片。

**根本原因**：
- `mattingPreviewUrls` 是通过监听 `mattingResults` 状态生成的
- 但批量抠图函数 `handleBatchSetMatting` 设置的是 `mattingPreviews` 状态
- 两个状态没有关联，导致 `mattingPreviewUrls` 为空数组
- `getPreviewUrl` 函数无法获取到正确的预览URL

## 实现步骤

### 步骤1：删除单个抠图功能相关代码

**在 ExportPanel.tsx 中删除以下内容：**

1. **删除状态和 Ref**
   - `activeSliceIndex` 状态
   - `isSelectingBounds` 状态
   - `tempSelection` 状态
   - `selectionStartRef` ref
   - `canvasRef` ref

2. **删除函数**
   - `handleSliceClick`
   - `handleMouseDown`
   - `handleMouseMove`
   - `handleMouseUp`
   - `cancelSelection`

3. **删除 UI 相关代码**
   - 区域选择遮罩层 (`selection-overlay`)
   - 选择画布 (`selection-canvas`)
   - 选择矩形 (`selection-rect`)
   - 取消选择按钮 (`selection-cancel-button`)
   - 单个格子的添加抠图按钮 (`matting-add-button`)

4. **简化 `handleClearSingleMatting`**
   - 保留此功能，允许用户清除单个已抠图的结果

### 步骤2：修复预览显示问题

**添加 useEffect 监听 `mattingPreviews` 变化：**

```typescript
useEffect(() => {
  if (mattingPreviews.length > 0) {
    const urls = mattingPreviews.map((p) => URL.createObjectURL(p.blob));
    setMattingPreviewUrls(urls);
    return () => {
      urls.forEach((url) => URL.revokeObjectURL(url));
    };
  } else {
    setMattingPreviewUrls([]);
  }
}, [mattingPreviews]);
```

**更新 `getPreviewUrl` 函数：**
- 简化逻辑，直接从 `mattingPreviews` 中获取对应的预览URL

### 步骤3：简化网格项渲染逻辑

删除单个选择相关的条件渲染，只保留：
- 图片预览
- 抠图指示器（显示/移除按钮）
- 下载按钮

### 步骤4：更新 CLAUDE.md 文档

记录本次修改内容。
