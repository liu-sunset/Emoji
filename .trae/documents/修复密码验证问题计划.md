# 修复计划：密码验证问题

## 问题描述
前端上传图片、输入提示词后点击生成，返回 `{"success":false,"error":"密码错误","code":"INVALID_PASSWORD"}`

## 根本原因
1. 前端登录组件（Login.tsx）验证密码为 "emo123" 后，仅将登录时间存储到 localStorage
2. 前端 API 调用（api.ts）发送请求时**没有携带 Authorization header**
3. 后端（local-server.js / index.js）期望从 Authorization header 获取密码进行验证
4. 前端和后端各自验证密码，导致密码验证不连通

## 修复步骤

### 1. 修改前端登录组件（Login.tsx）
- 登录成功后，将密码保存到 localStorage
- 添加 `savePassword(password)` 函数

### 2. 修改前端 API 工具（api.ts）
- 添加 `getPassword()` 函数从 localStorage 获取保存的密码
- 在 `generateImage` 函数的 fetch 请求中添加 `Authorization` header
- 格式：`Authorization: Bearer ${password}`

### 3. 修改后端 local-server.js（用于本地测试）
- 确认密码验证逻辑正确

### 4. 验证修复
- 使用 Chrome DevTools MCP 工具进行调试
- 启动本地后端服务
- 启动前端开发服务器
- 测试完整流程
