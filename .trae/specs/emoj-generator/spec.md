# 表情包生成器 - 功能规范

## 1. 概述

### 项目名称
EMO.J - AI表情包生成器

### 项目描述
一个基于阿里云百炼API的Web应用，用户上传图片后自动生成9宫格Q版贴纸表情包，支持一键剪切导出。

### 目标用户
- 社交媒体用户
- 需要快速制作表情包的非专业设计师

---

## 2. 技术架构

### 前端技术栈
- **React 18** + Vite构建工具
- 使用frontend skill优化界面设计
- 组件库：使用shadcn/ui或自定义圆角组件
- 状态管理：React hooks（useState/useContext）

### 部署目标
- 前端：静态托管（云开发CloudBase或COS）
- 后端：腾讯云云函数SCF

## 3. 功能模块

### 3.1 认证模块

#### 密码访问控制
- 用户首次访问时需输入访问密码
- 密码由管理员配置在云函数环境变量中
- 密码错误时显示错误提示，限制重试
- 登录成功后Session存储登录状态

**会话管理：**
- 使用LocalStorage存储登录状态
- 登录状态有效期24小时

### 3.2 图片上传模块

#### 支持格式
- JPG/JPEG
- PNG
- WebP
- 最大文件大小：5MB

#### 上传交互
- 拖拽上传区域
- 点击选择文件
- 上传后预览图片
- 支持重新上传替换

### 3.3 AI生成模块

#### 默认提示词
```
制作一个 10:16 比例的 3D 卡通风画布，画面包含九个 Q 版贴纸,每行排放3个，贴纸有着不同的穿搭、姿势和表情。请参考附带的图片进行创作。每个贴纸都带有白色边框，并且配有对话气泡，气泡内文字包括："早上好"、"午饭啦"、"咦"、"抱抱"、"谢谢"、"晚安"、"你超棒的"、"想你"、"亲亲😙"、"做得好" 以及 "保重"。背景采用柔和的白色到浅蓝渐变效果，营造出活泼、积极的氛围。
```

#### 气泡文字分配
- 从11个气泡文字中随机选择9个分配到9个贴纸位置
- 每个贴纸的气泡文字在生成时随机确定

#### 用户可修改部分
- 提示词的主体内容（穿搭、姿势、表情等描述）
- 气泡文字内容（可增删修改）

#### 用户不可修改部分
- 图片比例：10:16（固定值1120×1440）
- 贴纸数量：9个（3×3布局）
- 每行数量：3个

#### API调用限制
- 每个IP/会话每天限制生成10次
- 超出限制后提示用户明日再来

### 3.4 结果展示模块

#### 生成状态
- 加载中显示动画
- 成功显示生成结果图片
- 失败显示错误信息和重试按钮

#### 图片预览
- 原始比例展示
- 支持放大/缩小查看细节

### 3.5 剪切导出模块

#### 剪切算法
- 将10:16图片均分为9份
- 每份比例：10:16 ÷ 3 = 3.33:5.33
- 输出9张独立PNG图片

#### 导出功能
- 一键下载全部9张图片（ZIP打包）
- 支持单独点击某一张下载

## 4. 界面设计

### 整体风格
- 简洁清爽

- 移动端：单栏垂直布局

## 5. API设计

### 前端→后端

#### 生成图片
```
POST /generate
Content-Type: application/json

{
  "image": "base64编码的图片数据",
  "prompt": "修改后的提示词"
}

Response:
{
  "success": true,
  "image_url": "https://..." // 阿里云返回的图片URL
}
```

### 错误响应
```json
{
  "success": false,
  "error": "错误描述",
  "code": "ERROR_CODE"
}
```

### 错误码
- `INVALID_PASSWORD`: 密码错误
- `QUOTA_EXCEEDED`: 今日额度已用完
- `API_ERROR`: API调用失败
- `INVALID_IMAGE`: 图片格式或大小不符合要求

---

## 6. 安全考虑

### 后端转发
- API Key仅存储在云函数环境变量中
- 前端不暴露任何敏感信息

### 防滥用
- 访问密码保护
- API调用频率限制（每IP每天10次）

### 数据保护
- 不存储用户上传的原始图片
- 不存储生成的图片URL（由阿里云管理）
- 仅记录调用次数用于频率限制
