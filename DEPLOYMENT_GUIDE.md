# EMO.J 腾讯云部署指南

## 🔧 云函数 SCF 部署步骤

### 第1步：启用公网访问

在云函数配置界面，找到 **函数URL配置** 部分：

1. **公网访问** → 勾选 **启用**
2. **内网访问** → 保持 **启用**（可选）
3. **授权类型** → 选择 **开放**
4. 点击 **保存** 按钮

保存后，你会看到一个 **公网访问URL**，类似于：
```
https://service-xxxxxx.gz.apigw.tencentcs.com/release
```

**复制这个URL**，后面前端配置需要用到！

### 第2步：配置环境变量

在云函数控制台：

1. 进入 **函数管理** → **函数配置**
2. 找到 **环境变量** 部分
3. 点击 **编辑**
4. 添加以下环境变量：

| 变量名 | 变量值 |
|-------|-------|
| `DASHSCOPE_API_KEY` | 你的阿里云API Key |
| `ACCESS_PASSWORD` | emo123（或自行修改） |
| `DAILY_LIMIT` | 10 |

5. 点击 **保存**

### 第3步：测试云函数

1. 打开浏览器，访问你的云函数URL
2. 你应该看到返回的JSON数据，类似：
   ```json
   {"success":false,"error":"只支持 POST 请求","code":"INVALID_METHOD"}
   ```
3. 这表示云函数已成功部署！

## 🌐 前端 COS 部署步骤

### 第4步：创建存储桶

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 搜索 **对象存储 COS**
3. 点击 **存储桶列表** → **创建存储桶**

配置：
- **名称**：填写唯一名称（如 `emo-j-yourname`）
- **地域**：选择与云函数相同的地域
- **访问权限**：选择 **公有读私有写**
- **其他选项**：保持默认

点击 **创建**

### 第5步：配置前端API地址

1. 进入 `emo-j` 文件夹
2. 创建 `.env.production` 文件
3. 填写内容：

```env
VITE_API_BASE_URL=https://service-xxxxxx.gz.apigw.tencentcs.com/release
```

**注意**：替换为你的云函数URL！

### 第6步：构建前端项目

打开命令行：

```bash
# 进入项目目录
cd d:\thought\emoj\emo-j

# 安装依赖（如已安装可跳过）
npm install

# 构建项目
npm run build
```

构建完成后，会生成 `dist` 文件夹。

### 第7步：上传到 COS

1. 进入你创建的存储桶
2. 点击 **上传文件**
3. 选择 `dist` 文件夹中的所有文件
4. 点击 **上传**

### 第8步：配置静态网站

1. 进入存储桶 → **基础配置**
2. 找到 **静态网站配置** → **编辑**
3. 开启静态网站功能
4. **索引文档** 填写 `index.html`
5. 点击 **保存**

### 第9步：访问网站

在存储桶 **概览** 页面，找到 **访问域名**，类似于：
```
https://emo-j-xxxxxx.cos-website.gz.tencentcos.cn
```

**访问这个地址**，你的网站就可以使用了！

## 🔧 常见问题

### 1. 云函数返回 502 错误
- **检查**：环境变量 `DASHSCOPE_API_KEY` 是否配置正确
- **解决**：重新配置环境变量，确保API Key正确

### 2. 前端无法访问API
- **检查**：云函数的公网访问是否启用
- **检查**：前端的 `VITE_API_BASE_URL` 是否正确

### 3. 提示"密码错误"
- **检查**：云函数的 `ACCESS_PASSWORD` 环境变量是否配置
- **检查**：前端输入的密码是否正确

### 4. 图片上传失败
- **检查**：COS 的访问权限是否为"公有读私有写"
- **检查**：云函数是否正常运行
